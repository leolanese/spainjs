What are we going to talk about?


  • Since 2010, we've entered into a new phase in web design and development. The Responsive idea was conceived to solve a problem.
    Technology always evolves, trends always change. Now websites grow in their ambition and complexity, we are not longer building pages, we are building dynamic and interactive Responsive web applications.


  • We are going to share advanced and emerging techniques for improving Responsive applications. Focusing on "The Good Parts", avoiding the classic pitfalls, with the emphasis on latest trends and revealing what RWD really can do for you.
@leolaneseltd

Responsive Web Design:

The
Good
Parts

Simply put: Responsive Web Design is:

  • Flexible, grid-based layout
  • Flexible media: images; and video;
  • @media queries

RWD is just the beginning.


"The Good Parts" are on top of it:

  • Mobile First
  • Progressive Enhancement
  • Automatic Tests
"Responsive Web Design is one of the most important technologies to appear in recent years. But using RWD badly may be worse than not offering it at all", Craig Buckler

a better RWD definition


Responsive Web Design is the practice of creating reliable future friendly websites; that fluidly respond to deliver the right user experience for the right environment at the right time.

Case study: RWD video

  • Video (HTML5 video)
    • - external embed videos needs to use <iframe> instead <video>
  • Video (iframe + intrinsic ratio)
    • - can't dynamically load external videos because you don't know how the aspect-ratio would be
  • Video (iframe + FitVids.js)
    • - embed videos using <iframe> instead <video>
    • + create a wrapper around the iframe to preserve the aspect ratio.

Case study: Performance

  • Mobile First development and Mobile First Design
    • Treat Performance As Design
    • Focus only the most important data
  • Progressive Enhancement:
    • JS as a feature: js no-js; or hasClass; or clientInformation;
    • Feature detection: Modernizr and has.js
      • Targeting DOM Objects navigator.connection.type; clientInformation; IE10; Chrome27; full list.
      • Targeting JavaScript API matchMedia; or enquire.js ECMAScript5/6; 'use strict'; JSON parsing; (Thanks Douglas Crockford !)
      • Targeting CSS CSS @support; Modernizr + @support;
    • Conditional loading: Yepnope; @media queries image as conditional;
    • Dynamic function loading (Lazy Loading)
"As web creators we need to see performance as design", @elementByIdea
